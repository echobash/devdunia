THE FOUR REALMS

Where your code actually lives — read left to right!

📂
REALM 1
Working Dir
your local filesystem
The wild, untamed frontier. Every file you can see, edit, and break lives here. Nothing is safe until you snapshot it!
"I'm where the magic HAPPENS! Also where you accidentally delete things..."
🧳
REALM 2
Staging Area
.git/index
The green room. You carefully choose which changes get into the next commit. Director's cut only — nothing sneaks through!
"I'm the bouncer. Not everything from working dir gets past me!"
💾
REALM 3
Local Repo
.git/objects/
Your personal time machine. Every commit is a permanent snapshot. The full history of your project lives right here on your disk.
"I remember EVERYTHING. Every commit, every branch, every mistake."
☁️
REALM 4
Remote Repo
origin / github.com
The shared universe! GitHub, GitLab, Bitbucket — where teams sync up, open PRs, and the whole world can see your masterwork.
"I'm the cloud fortress. Push to me when you want teammates to see your work!"
🧳
BONUS REALM
The Stash
refs/stash
The junk drawer! Unfinished work you're not ready to commit goes here. Run away, save state, come back later. It's the panic button.
"Boss asked for a hotfix mid-feature? STASH IT! I'll hold your mess."
🔍
SECRET REALM
The Reflog
.git/logs/HEAD
Git's secret diary. Every movement of HEAD is logged here. Accidentally deleted a branch? Forced a reset? Reflog is your undo button.
"They think I'm invisible. But I see ALL. git reflog will save your life someday."
COMMIT EARLY, COMMIT OFTEN!     NEVER FORCE-PUSH MAIN!

THE CAST OF COMMANDS

Nine legendary powers — click a card to flip it!

COMMIT SNAPSHOT
The Chronicler — takes a permanent, immutable snapshot of everything in staging. Each commit is a node in your timeline, forever.
$ git commit -m "your message"
► FLIP TO READ!
COMMIT — PRO TIPS
Write meaningful messages! Future-you (and teammates) will thank you.

git commit --amend fixes the last commit message.
git commit -a -m "msg" stages tracked files automatically.

Each commit stores: tree snapshot, author, timestamp, parent hash.
BRANCH POINTER
The Reality Splitter — a branch is just a lightweight pointer to a commit. Create universes for free, merge them back when done!
$ git branch feature/login
► FLIP TO READ!
BRANCH — PRO TIPS
Branches are free! They're just 41-byte files pointing to a commit hash.

git branch -d name deletes a merged branch.
git branch -D name force-deletes (careful!).
git branch -a shows all including remotes.
MERGE COMBINE
The Reunifier — brings diverged branches together, weaving timelines into one. Creates a merge commit with two parents. History preserved!
$ git merge feature/login
► FLIP TO READ!
MERGE — PRO TIPS
Fast-forward merge happens when there's no divergence — no merge commit needed!

git merge --no-ff forces a merge commit even when fast-forward is possible.
git merge --abort bails out during a conflict.
REBASE REWRITE
The Time Traveler — replays your commits onto a new base. Creates a CLEAN linear history. But beware: rewrites history, never on shared branches!
$ git rebase main
► FLIP TO READ!
REBASE — PRO TIPS
Golden rule: Never rebase public history!

git rebase -i HEAD~3 interactive rebase — squash, reword, drop commits.
git rebase --abort cancels mid-rebase.
git rebase --continue after resolving conflicts.
CHERRY-PICK PLUCK
The Surgeon — applies one specific commit from any branch onto your current one. No need for a full merge, just grab the commit you want!
$ git cherry-pick a1b2c3d
► FLIP TO READ!
CHERRY-PICK — PRO TIPS
Use case: hotfix was committed to wrong branch? Cherry-pick it to the right one!

git cherry-pick A..B picks a range of commits.
Creates a NEW commit with a different hash but same changes.
RESET UNDO
The Eraser — moves HEAD backward in time. Three modes: soft (keep changes staged), mixed (unstage), hard (nuke everything). Powerful. Dangerous.
$ git reset --soft HEAD~1
► FLIP TO READ!
RESET — PRO TIPS
--soft: move HEAD, keep index + working tree unchanged.
--mixed: move HEAD + unstage, keep working tree (DEFAULT).
--hard: move HEAD + destroy all uncommitted changes.

git reset --hard is irreversible (unless you use reflog!).
STASH SHELVE
The Pause Button — saves your messy work-in-progress without a commit so you can switch branches with a clean slate. Resume later!
$ git stash push -m "wip login"
► FLIP TO READ!
STASH — PRO TIPS
git stash list — see all stashed states.
git stash pop — apply newest stash + delete it.
git stash apply stash@{2} — apply specific stash, keep it.
git stash branch new-branch — turn a stash into a branch!
LOG HISTORY
The Historian — shows the commit timeline. Pair with flags to get graphs, one-liners, file diffs, or range comparisons. The most powerful viewer!
$ git log --oneline --graph --all
► FLIP TO READ!
LOG — PRO TIPS
git log -p show full diffs per commit.
git log --author="Ali" filter by author.
git log --since="2 weeks ago" filter by date.
git log main..feature commits in feature not in main.
git log --follow path/to/file follow file renames.
BISECT DEBUG
The Detective — binary-searches your commit history to find exactly which commit introduced a bug. Cuts the search space in half every step!
$ git bisect start
► FLIP TO READ!
BISECT — PRO TIPS
Flow:
git bisect start
git bisect bad (current is broken)
git bisect good v1.2.3 (this was fine)
Git checks out midpoints, you mark good/bad.
git bisect reset when done. O(log n) bug hunting!
MERGE CONFLICTS? STAY CALM!     GIT BISECT SAVES HOURS!
3
THE GREAT JOURNEY

Follow a feature from idea to merged PR — the complete adventure!

1
"THE CALL TO ADVENTURE!"
A new feature is born! Clone or pull the latest main, then spin up a fresh branch. Never work directly on main — that's how legends die.
$ git pull origin main  $ git checkout -b feature/add-login
2
"THE FORGE OF CREATION!"
Write code. Break things. Fix them. Check what changed at any point with git status and git diff. Stay aware of your working directory at all times.
$ git status  $ git diff
3
"STAGE YOUR CHAMPIONS!"
Add only the changes that belong together in one logical commit. Use interactive staging to be surgical — don't just "git add ." blindly!
$ git add src/login.js  $ git add -p
4
"CAPTURE THE MOMENT!"
Commit with a clear, present-tense message. Describe WHY, not just WHAT. Your future teammates will either thank you or curse you here.
$ git commit -m "add JWT-based login endpoint"
5
"STAY CURRENT, HERO!"
Main has moved while you worked. Rebase your feature branch onto the latest main to keep a clean linear history before opening your PR.
$ git fetch origin  $ git rebase origin/main
6
"PUSH TO THE CLOUD FORTRESS!"
Send your branch up to the remote so teammates can review it. First push requires -u to set the upstream tracking reference. Subsequent pushes are simple.
$ git push -u origin feature/add-login
7
"THE COUNCIL REVIEWS!"
Open a Pull Request. Reviewers comment, CI runs tests, linters scold you. Address feedback with new commits. A great PR description is half the battle.
$ gh pr create --title "Add login" --body "..."
★
"MERGED! GLORY AT LAST!"
The PR is approved and merged to main. Delete the remote branch, pull main locally, and delete the local branch. The feature is alive in production. You are victorious!
$ git checkout main && git pull  $ git branch -d feature/add-login
THE END!
...until the next feature branch is born. Then our hero codes again!
